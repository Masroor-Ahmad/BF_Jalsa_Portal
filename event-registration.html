🛠️ Tools
📱 Self-Registration QR Code

Display this QR code at your event entrance or on a desk screen. Guests scan it with their phone to register themselves, or use a shared tablet in kiosk mode.

⚠️ This page is opened as a local file (file://…). Guests on other devices cannot reach it via QR code. To enable multi-device access, serve this file via a local web server or host it online. See instructions below.
How guests use this:
1. Guest scans QR with their phone → fills in registration form → taps Submit
2. Their phone shows a Registration Code (QR)
3. Staff at desk taps Admin → Import Guest Code and scans the guest's phone screen

To use on the same tablet (kiosk mode):
Tap "Open Kiosk Mode" below — the screen becomes a clean guest-facing form with no admin access. Exit with your PIN.

To serve on local Wi-Fi (guests use own phones):
Open Terminal and run: python3 -m http.server 8080 in the folder containing this file.
Then your QR URL will be http://[your-ip]:8080/event-registration.html?kiosk

⚙️ Event Settings
🔥 Live Sync — Firebase Not configured

Configure Firebase so guests registering on their own phones appear instantly on this system — no QR handoff needed. Free to set up (takes ~2 minutes). Needs Wi-Fi at the venue.

📋 How to set up Firebase (free, 2 minutes)
1. Go to console.firebase.google.com and sign in
2. Click Add project → give it any name → Continue
3. In the left menu: Build → Realtime Database → Create Database
4. Choose a region, then select "Start in test mode" → Enable
5. In the left menu: Project settings (gear icon) → scroll down to "Your apps" → click the Web icon (</>)
6. Register the app (any nickname) → copy the apiKey, projectId, and databaseURL values into the fields above
7. Paste the same config into every device (staff laptops, hall tablets) that needs to see registrations
Note: Test mode is open for 30 days. For longer events, set a rule: { "rules": { ".read": true, ".write": true } }
